  • What is the purpose of HTML5 semantic tags? Which ones do you see as most useful?
    • The purpose of HTML5 semantic tags are to give meaning and structure to content. They allow a developer to give meaning to a web page and better structure a web page for both humans and machines. Non-semantic elements like div and span provide no inherent meaning, whereas form, table, and img tags provide meaning and intent. HTML5 introduced new elements such as article, aside, and nav gives more semantic meaning to a website.
  • What is the order the browser uses to display elements on a page?
    • The rendering engine of the browser first parses the HTML to construct the Document Object Mode tree (DOM). The browser will turn the tags of the HTML document into DOM nodes called the "content tree". the rendering engine will also parse the external CSS. It will then combine the styling information with the HTML to create a render tree. After building the render tree, the browser will then go through a "layout" process. The final stage is the painting stage.
